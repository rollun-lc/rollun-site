# Интент: сайт Rollun (Next.js + Payload CMS)

## 1. Проект

Сайт компании Rollun на **Next.js (App Router) + TypeScript + React**, воссоздающий handoff-дизайн **пиксель-в-пиксель**, с контентом, редактируемым менеджерами через **Payload CMS**. Матчим визуал дизайна, а не копируем DOM прототипа. Данные тянутся из Payload через local API прямо в React Server Components (тот же процесс, без сетевого хопа).

## 2. Приоритеты

| # | Приоритет | Статус |
|---|-----------|--------|
| 1 | **Дизайн пиксель-в-пиксель** | Главная ценность, релизится первым |
| 2 | CMS / админка для менеджеров | Вторично — «на будущее, на закос» |

Сайт **некритичный**. Менеджер сам разберётся с админкой. CMS не должна блокировать дизайн-релиз.

## 3. Стек и ключевое архитектурное решение

- **Payload CMS 3.0** — TS-native, работает **внутри того же Next.js App Router** (local API, self-host на инфре rollun, opensource). Авто-генерит админку, встроенная Media-коллекция с авто-ресайзом/оптимизацией, access control.
- **Ключевой ход:** «**захардкоженный layout + типизированные CMS-слоты**». Вёрстка захардкожена в React; CMS заполняет только заранее размеченные поля (hero-картинка, заголовок блока, текст). Менеджер физически не может сломать дизайн.
- Одно решение снимает три напряжения разом: дизайн 1:1 не ждёт CMS, менеджер не ломает вёрстку, MVP быстрее.
- **next/image** для картинок. **On-demand revalidation / ISR** — правки менеджера появляются быстро без пересборки.

## 4. Модель контента

**Globals:**
- `SiteSettings` — паспорт компании (тел/адрес/GitHub/LinkedIn/часы работы). Тянется во все страницы, шапку, футер, Contact, Our Shops. Правка в одном месте → обновилось везде.
- По одному Global на каждую из 6 страниц: `HomeContent`, `AboutContent`, `CatalogContent`, `BrandsContent`, `ShopsContent`, `ContactContent` — поля ровно под редактируемые слоты страницы (не абстрактная «Page», нельзя сломать структуру).

**Collections:**
- `Products` — с полем-швом `sku`/`externalId` уже сейчас (ручной ввод), чтобы будущий автосинк дописывал цену/картинку/наличие поверх без переделки модели.
- `Brands`, `Shops`/`Locations`.
- `Media` — авто-оптимизация (Payload + next/image режут тяжёлые картинки в webp/нужные размеры).
- `Posts` — на будущее.
- `Users` — встроенный auth Payload для менеджеров.

## 5. Что редактируемо vs гвоздями

| Уровень | Что | В CMS? |
|---------|-----|--------|
| 🔴 Живое | Картинки, товары | Да |
| 🟡 Правимое | Тексты блоков, живые надписи | Да |
| 🟢 Паспорт | Тел / адрес / GitHub / LinkedIn / часы | Да (SiteSettings) |
| ⚫ Гвоздями | Вёрстка, структура, юр/статичный текст, микрокопи | Нет, в коде |

Граница текста: живые надписи — через CMS; юр/статичный текст — гвоздями в коде.

## 6. Дизайн-фиделити (приоритет №1)

- **DS-токены** из `colors_and_type.css` портируются **1:1** в CSS-переменные / Tailwind theme: оранж `#EA7B08`, charcoal, spacing / radii / shadows / easing.
- **Шрифты:** Archivo, Hanken Grotesk, Spline Sans Mono (Google Fonts, как в DS).
- **6 desktop + Mobile-версии** страниц из handoff-бандла: `rollun_handoff/rollun-web-site/project/`. Каждый HTML-прототип пересобирается в React-компоненты пиксель-в-пиксель; Mobile-версии HTML → адаптив-брейкпоинты.

## 7. Роадмап

| Фаза | Содержание |
|------|-----------|
| **0** | Скаффолд Next.js + Payload в одном проекте, порт DS-токенов и шрифтов, базовый layout (шапка/футер) |
| **1** (главная ценность) | Все 6 страниц пиксель-в-пиксель с **захардкоженным** контентом. Это и есть «дизайн 1:1» — релизится первым, без CMS |
| **2** | `SiteSettings` Global (паспорт) + Media-коллекция; перевод контактов/ссылок/картинок на CMS-слоты |
| **3** | `Products`/`Brands`/`Shops` коллекции; Catalog и Our Brands из CMS, ручной ввод карточек |
| **4** (на будущее, «закос») | Авто-подтяг товаров по sku-шву из внешнего фида; Posts/новости |

## 8. Явные scope-решения

- **Авто-подтягивание товаров — НЕ сейчас.** Желательно в перспективе (модель к нему готова через sku-шов), но в текущем скоупе — **ручной ввод** карточек.
- Сайт **некритичный**; MVP-фокус на дизайне, CMS подключается слоями потом.
