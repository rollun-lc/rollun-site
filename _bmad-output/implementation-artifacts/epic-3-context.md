# Epic 3 Context: Главная страница (Home)

<!-- Generated from planning artifacts. Regenerate with compile-epic-context if planning docs change. -->

## Goal

Эпик реализует главную страницу сайта (`/`) — точку входа оптового B2B-партнёра, который за секунды должен считать, что Rollun — живая технологичная компания. Страница собирается пиксель-в-пиксель по Handoff (desktop + mobile) со всеми ключевыми анимациями: hero-мозаика оживает, счётчики «Proven at scale» досчитывают цифры масштаба, продуктовые линии листаются вручную, а кнопка «GET IN TOUCH» открывает Contact-форму. Это первая полностраничная реализация после оболочки и формы — она задаёт паттерн «страница = чистая функция типизированного контент-объекта» и доказывает, что связка «оболочка (Epic 1) + форма (Epic 2)» собирается в готовую страницу с пиксель-приёмкой.

## Stories

- Story 3.1: Каркас Home — композиция и статический контент
- Story 3.2: Hero-мозаика с bloom-анимацией
- Story 3.3: Переключатель продуктовых линий
- Story 3.4: Счётчики «Proven at scale» и маркетплейс-карточки

## Requirements & Constraints

- Главная рендерится пиксель-в-пиксель по Handoff на desktop и mobile; отрисованный прототип — источник истины, известные дефекты воспроизводятся дословно (не «чинятся»). В частности: mobile-версия секции «Find us on marketplaces» показывается без рейтингов маркетплейсов — это дефект дизайна, оставляется как есть.
- Секции Home идут строго в порядке Handoff: Hero → Product lines (Automotive/Health) → Stats «Proven at scale» → Key benefits → «Find us on marketplaces» → CTA «Let's talk business» → Footer.
- Приёмка страницы формально гарантируется чеклистом пиксель-приёмки (введён в Epic 1): страница не Done, пока не отмечены все её брейкпоинт-пункты (постраничные desktop-наборы вплоть до 1280 + mobile-композиция ≤768px, летербокс-шелл 440px). Воспроизводимые дефекты в чеклисте помечены как «ожидаемо».
- Все анимации уважают `prefers-reduced-motion` — при включённом режиме показывается статичный финальный кадр/значение, без движения.
- Базовый a11y-пол: семантический HTML, корректный Tab-порядок, видимый оранжевый focus-ring, `prefers-reduced-motion`. A11y не меняет отрисованный пиксель — при конфликте приоритет у дизайна.

## Technical Decisions

- **Страница = чистая функция типизированного контента (AD-7).** Home — чистая функция типа `HomeContent`. В Фазе 1 подаётся статический инстанс из `content/*`, typecheck-совместимый с будущим Payload Global (шов к Epic 7, менять его не нужно). Клиентского фетча контента нет.
- **Islands Architecture (AD-1, AD-4).** Секции — RSC; интерактив/анимации (hero-bloom, переключатель линий, count-up) — листовые `'use client'`-островки, получающие весь контент пропсами. Островок никогда не фетчит контент сам.
- **Две композиции desktop|mobile (AD-3).** Обе SSR-рендерятся в DOM, переключение — только CSS-медиа на 768px; JS-гейтинг и UA-сниффинг запрещены. Тяжёлые hero-ассеты (15–18 МБ) не дублируются в оба дерева — art-direction подаёт ровно одну композицию картинок на вьюпорт.
- **Мозаика hero (UX-DR19).** Desktop: ч/б плитки, ~25% зажжены в цвет, набор обновляется каждые 3с (CSS-переход 1.6s). Mobile: одна фотография «выезжает» цветной из плитки по таймингам MOVE 1000 / HOLD 3200 / GAP 800 мс, затем следующая.
- **Переключатель линий (UX-DR11).** Desktop: кнопки-цифры `.line-dots` тогглят слайд (active/prev/next), **без авто-ротации**, синхрон с URL hash. Mobile: горизонтальный `scroll-snap`-шелф (слайды 78%), свайп/scroll-snap, без переключателей.
- **Count-up счётчики (UX-DR18).** Секция Stats «Proven at scale»: старт по IntersectionObserver threshold 0.4, cubic ease-out ~1800ms, форматирование comma/suffix, играют ровно один раз; до триггера финальное значение не показано; reduced-motion → сразу финал.
- **Contact-форма (переиспользование Epic 2).** Кнопка «GET IN TOUCH» монтирует существующий `ContactForm`: desktop — модалка, mobile — навигация на `/contact` (mobile-модалки нет). Не реализуется заново в этом эпике.
- Микрокопи и CTA — англоязычные, «гвоздями» в коде (капсом, глаголом: «GET IN TOUCH», «Let's talk business»); в код, а не в контент-слот.

## UX & Interaction Patterns

- **Reveal-on-scroll:** все секции кроме первой получают `.in` по IntersectionObserver (threshold desktop 0.12 / mobile 0.1); desktop-фолбэк форсит видимость в вьюпорте через 1.5с (общий паттерн из Epic 1).
- **Empty / no-data:** плейсхолдеры вида «Photo N» до реальных фото; неподгруженное изображение прячется через `onerror`, не ломая вёрстку.
- **Клавиатура и фокус:** все интерактивные элементы доступны с клавиатуры, видимый оранжевый focus-ring (`outline:2px solid #EF7F1A`, offset 3px), ARIA на оверлеях (модалка формы — `role=dialog` + focus-trap + Esc).

## Cross-Story Dependencies

- **Epic 1 (оболочка + DS + мобильное шасси):** Home использует общие header/footer, дизайн-токены `styles/theme.css`, шрифты `next/font`, мобильный drawer/scroll-lock и паттерн reveal-on-scroll. Пиксель-приёмка страницы опирается на чеклист из Story 1.6.
- **Epic 2 (ContactForm):** кнопка «GET IN TOUCH» монтирует готовый компонент формы и его режимы показа (desktop-модалка / mobile-переход на `/contact`).
- **Внутри эпика:** Story 3.1 (каркас + `HomeContent` + монтаж CTA) — фундамент для островков в Stories 3.2–3.4; секция маркетплейс-карточек делит стори 3.4 со счётчиками Stats.
- **Шов к Epic 7 (Фаза 2):** статический `content/*`-инстанс позже заменяется одноимённым Payload Global `HomeContent` без изменения разметки/пикселя.
